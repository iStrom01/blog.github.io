ThreadLocal 原理

每个线程都维护了一个 ThreadLocalMap，其 key 指向 ThreadLocal

ThreadLocalMap 没有实现 Map 而是独立实现了 map 的功能，内部 entry 也是独立实现的

虽然 Key 是弱引用，但是 ThreadloaclMap 是 Thread 的一个属性，value 是强引用，生命周期和线程一样长，无论强引用弱引用只要不手动删除就可能内存泄露

ThreadLocalMap 中 set/getEntry 中会对key 为 null 进行判断，如果 key 为 null，会对 value 置空
